INCOHÉRENCES

Les boutons suivants sont des <button> et devraient hériter du pixel ochre
via la règle globale "a, button, .clickable" dans flutgraben.css,
mais sont écrasés par cursor: pointer (natif) dans interior.css :

  - .back-to-top
  - .subnav-item
  - .archive-year
  - .residents-search-clear
  - #lightbox-prev
  - #lightbox-next

Seul #lightbox-close a le pixel ochre correctement, car il le redéfinit explicitement.

details > summary reçoit pointer natif — cohérent visuellement
mais pas le pixel. summary n'étant pas un <button>, la règle globale
ne s'applique pas.


CURSEURS DISPONIBLES

  Pixel rust  — défaut, fond de page, zones non interactives
               fill: #573835 (var --rust), stroke: #e5e9ed (var --bg)

  Pixel ochre — éléments interactifs (liens, boutons)
               fill: #c17d4c (var --ochre), stroke: #e5e9ed (var --bg)

  zoom-in     — images cliquables (.row-photo img)
               curseur natif du navigateur

  default     — .gif-anchor (reset volontaire sur le conteneur)
